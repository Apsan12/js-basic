JavaScript for Beginners — Quick Guide

1) What is JavaScript?
- JavaScript (JS) is a programming language used to make web pages interactive and to build servers (with Node.js).
- Runs in two main environments: the browser (has DOM, window, document) and Node.js (server-side, no DOM).

2) How to run the examples
- In the browser: create an HTML file, include a <script> tag, open in a browser.
- In Node.js: save code to a .js file and run `node filename.js` from a terminal.

3) Basic concepts and examples
- Comments
	// single-line comment
	/* multi-line comment */

- Variables (modern JS uses let and const)
	let count = 5;        // changeable
	const pi = 3.14;      // constant

- Data types
	Number, String, Boolean, null, undefined, Object, Array, Function

- Example: printing and functions
	console.log('Hello, World!');

	function greet(name) {
		return `Hello, ${name}!`;
	}
	console.log(greet('Alice'));

- Example: conditionals
	const age = 20;
	if (age >= 18) {
		console.log('You are an adult.');
	} else {
		console.log('You are a minor.');
	}

- Example: loops
	for (let i = 0; i < 5; i++) {
		console.log('Iteration: ' + i);
	}

- Arrays and objects
	const fruits = ['Apple', 'Banana', 'Cherry'];
	const person = { name: 'Bob', age: 30 };
	console.log(fruits[0], person.name);

4) Browser vs Node: DOM and document
- The `document` and `window` objects only exist in browsers. Running code that uses `document` directly in Node will cause `ReferenceError: document is not defined`.
- If you want browser code, put it in an HTML page or guard it:
	if (typeof document !== 'undefined') { /* DOM code */ }
- To run DOM-like code in Node for testing, use `jsdom` (npm package).

5) Small exercises (try these)
- Exercise 1: Create a file `hello.js` with console.log('Hello from Node'); run with `node hello.js`.
- Exercise 2 (browser): create `index.html` with a button and a script that shows an alert when clicked.
- Exercise 3: Write a function that takes an array of numbers and returns the sum.

6) Solutions (quick)
- Sum function
	function sum(arr) {
		let total = 0;
		for (let n of arr) total += n;
		return total;
	}

7) Quick tips and common pitfalls
- Use === and !== for comparisons (strict equality).
- Avoid re-declaring variables with let/const in the same scope.
- Use `console.log` to debug; in browsers you can also use DevTools (F12).

8) Next steps and resources
- Learn about: ES6 features (arrow functions, classes, destructuring), asynchronous JS (callbacks, promises, async/await), modules (import/export), and DOM manipulation.
- Recommended free resources: MDN Web Docs (javascript), freeCodeCamp, and the Node.js docs.

MERN Stack — Beginner-friendly step-by-step guide (with comments on each step)

1) What is MERN?
// MERN = MongoDB + Express + React + Node. Use JavaScript end-to-end.

2) Quick roadmap (high level)
// Start: JS fundamentals -> Node & npm -> Express -> MongoDB/Mongoose -> React -> Connect frontend+backend -> Auth -> Deploy

3) Detailed steps with comments
- Step A — Strong JavaScript foundation
	// Learn variables, types, functions, arrays, objects, classes, and ES6 features.
	// Practice: build small programs and use async/await for asynchronous flows.

- Step B — Node.js basics
	// Node runs JS outside the browser. Learn module system (require/exports or import/export), file I/O, and npm.
	// Practice: create simple CLI scripts and small servers using Express.

- Step C — Express.js (server framework)
	// Routing and middleware are core. Learn how to parse JSON and handle request/response.
	// Practice: create REST endpoints: GET /items, POST /items, PUT /items/:id, DELETE /items/:id.

- Step D — MongoDB + Mongoose
	// MongoDB stores JSON-like documents. Mongoose adds schemas and validation.
	// Practice: define schemas, connect to MongoDB, and implement CRUD operations.

- Step E — React (frontend)
	// Learn components, JSX, hooks (useState/useEffect), props, and state management basics.
	// Practice: create a small SPA that fetches data from your Express API.

- Step F — Connect front and back
	// Use fetch or axios from React to call Express API. Handle CORS in development.
	// Practice: build forms to create data on the server and display data returned by API.

- Step G — Authentication
	// Implement JWT-based auth or session auth. Learn hashing passwords (bcrypt) and protecting routes.

- Step H — Deployment
	// Learn environment variables and build scripts. Use services like Render, Heroku, Vercel, Netlify, or VPS.
	// Put database in MongoDB Atlas for production.

4) Small project ideas (practice with comments)
- Todo App (start simple): React frontend + Express + MongoDB backend.
	// Add features step-by-step: list tasks, add tasks, edit/delete tasks, mark complete.
- Blog App: users, posts, comments, authentication.
	// Shows relations, user ownership, and more advanced queries.
